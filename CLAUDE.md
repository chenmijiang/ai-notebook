# CLAUDE.md

本文件为 Claude Code (claude.ai/code) 在处理本仓库代码时提供指导。

## 仓库概述

这是一个文档仓库（ai-notebook），包含关于软件开发主题的中文技术指南和笔记。本仓库没有应用程序代码、构建系统或测试。

## 内容结构

本仓库由独立的 Markdown 文件组成，每个文件涵盖一个特定的技术主题：

- 技术指南使用中文编写
- 每篇指南都是包含示例的综合参考文档
- 文件命名反映其内容主题
- 文件存放在**docs/** 目录下

## 写作规范

在本仓库中创建或编辑文档时：

### 基本要求

- 使用简体中文编写内容
- 代码示例应简洁，聚焦于核心概念，避免冗长
- 使用清晰的标题和子标题组织内容
- 包含展示实际应用场景的实用示例
- 确保内容准确且易于理解，必要时需要`web fetch/search` tool 验证信息

### 文档结构

- **标题命名**：使用「主题 + 完全指南/使用指南」格式，如 `# Git Hooks 完全指南`
- **目录**：文档开头提供完整的目录，使用 markdown 锚点链接，可采用嵌套或简单列表格式。为了不要让目录过长，可只列出一级和二级章节
- **章节编号**：
  - 一级章节使用数字编号（1. 2. 3.）
  - 子章节使用点号分隔（1.1, 1.2, 1.1.1）
  - 标题格式：`## 1. 概述`、`### 1.1 什么是 XX`
- ~~**章节分隔**：主章节（一级章节）之间使用 `---` 分隔线~~，废弃，内容风格统一不使用章节分隔线
- **结尾部分**：
  - 「总结」章节：回顾核心要点，可包含速查表
  - 「参考资源」章节：提供官方文档和相关资源链接

### 内容格式

- **表格**：大量使用 markdown 表格进行对比说明、参数说明、速查总结
- **图解**：
  - 优先使用 ASCII 艺术绘制简单图解（如 Git 分支图、流程图）
  - 复杂图形可使用 Mermaid 语法（类图、时序图、流程图）
- **代码块**：
  - 标注语言类型（bash, json, jsonc, javascript 等）
  - 只包含必要的关键代码，避免冗长
  - 使用中文注释说明关键步骤
  - 在注释中使用 `// ✅` 和 `// ❌` 标记好/坏实践
- **提示信息**：使用引用块格式
  - `> **注意**：` - 重要警告或注意事项
  - `> **提示**：` - 有用的建议
  - `> 说明：` - 补充说明信息
- **对比说明**：使用 ✅ 和 ❌ 标记好/坏实践，可在正文或代码注释中使用
- **FAQ**：使用 `**Q1: 问题？**` 格式，后接解答段落或代码示例。问题应聚焦于技术实现、常见错误排查、配置疑难等技术内容
- **速查表**：在总结章节提供表格形式的快速参考

### 示例规范

- 提供完整可运行的代码示例
- 包含多个实际应用场景（场景1、场景2...）
- 代码示例中使用中文注释说明关键步骤
- 复杂操作提供分步骤说明（1. 2. 3.）
