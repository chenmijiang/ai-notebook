# curl 前端开发者实用指南 — 设计文档

## 目标

为前端开发者编写一篇精简实用的 curl 指南，帮助他们在日常 API 调试、接口测试中快速使用 curl。

## 读者画像

- 前端开发者，熟悉 HTTP 基础
- 需要在终端中快速调试 API，而非依赖 Postman 等 GUI 工具
- 不需要深入了解 curl 的所有选项

## 结构方案

采用**场景驱动**结构，围绕前端开发者的实际使用场景组织内容。

## 目录

1. **概述** — curl 是什么、前端为什么需要它、与 Postman/fetch 的定位区别
2. **基础请求** — GET 请求、带查询参数的 GET
3. **发送数据** — POST JSON、表单提交、PUT/PATCH/DELETE
4. **请求头与认证** — 自定义 Header、Bearer Token、Cookie
5. **文件上传与下载** — multipart 上传、下载文件
6. **调试技巧** — -v 详细输出、-o 保存响应、-w 查看耗时、只看响应头
7. **实用技巧** — 跟随重定向、设置超时、忽略 SSL 证书、使用代理

## 风格

- 每个场景以"你想做什么"开头，紧跟命令示例
- 示例使用 httpbin.org 或通用占位 URL
- 简短解释选项含义，不展开原理
- 中文编写，遵循项目 tech-docs-guide 规范
- 约 1500-2000 字
