# ESLint 系统学习设计方案

## 概述

为零基础但有 TypeScript + React 项目的开发者设计的 ESLint 专家级系统学习方案。

## 目标

- 达到专家水平：能独立配置项目、能解决实际问题、能编写自定义规则
- 时间：1-2周高强度学习

## 架构：6+1 结构

```
第1篇 → 第2篇 → 第3篇 → 第4篇 → 第5篇 → 第6篇
  ↓                               ↓
第7篇(新增) ← ← ← ← ← ← ← ← ← ← ←
```

## 7篇文档设计

### 第1篇：基础概念与原理

- ESLint 定位与价值
- 核心概念：Rules、Plugins、Extends、Environments、Parser
- Flat Config 基础用法（入门级）
- CLI 命令详解
- 工作原理：Parser → AST → Rules → Report

**检验标准**：能解释 ESLint 工作原理，能初始化基础配置

### 第2篇：规则系统详解

- 规则严重程度：off/warn/error
- 规则分类与查找
- 常用规则详解（按场景分类）
- extends 继承体系
- 插件使用与配置
- 规则选项配置

**检验标准**：能根据报错信息找到规则，能配置适合项目的规则集

### 第3篇：框架集成（TS/React/Vue）

- @typescript-eslint 配置
- 类型感知 linting（projectService）
- eslint-plugin-react + react-hooks
- Vue 项目配置
- Next.js / Vite 脚手架配置

**检验标准**：能为 TS + React 项目配置完整 ESLint

### 第4篇：团队工作流与工程化

- Git Hooks：husky v9 + lint-staged v15
- CI/CD 集成（GitHub Actions）
- ESLint + Prettier 协作
- 团队规范制定
- 共享配置 npm 包

**检验标准**：能搭建团队级 ESLint 工作流，能发布共享配置

### 第5篇：进阶定制

- Flat Config 深度（迁移、调试、配置继承）
- 自定义规则入门（AST、规则结构、RuleTester）
- 插件开发基础
- 性能优化（cache、timing）

**检验标准**：能编写自定义规则，能开发 ESLint 插件

### 第6篇：实战综合

- 完整项目配置案例
- 从零搭建实战
- 配置解读与理解
- 常见场景解决方案

**检验标准**：能独立为项目配置 ESLint，能解决 90% 配置问题

### 第7篇：问题排查与性能（新增）

- 规则不生效排查
- 配置冲突解决
- 性能诊断与优化
- Monorepo 配置
- 故障案例分析

**检验标准**：能诊断 ESLint 问题，能优化大型项目性能

## 关键改进点

1. **新增第7篇**：专门讲问题排查和性能，这是专家水平的必备技能
2. **消除重复**：每篇边界清晰，Flat Config 基础在第1篇，深度在第5篇
3. **检验标准**：每篇有明确的「能解决什么问题」清单
